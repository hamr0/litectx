## The stash() feature (R-C4 restorable compression) — what it is

- **`LiteCtx.stash(id, text) → void`** (`src/index.js`) + **`Store.writeStash`** + a new plain
  `stash(path, text, created_at)` table (NOT fts5) + `getItem` fallback to stash + `forgetMemory`
  handles a stash by id. 6 integration tests (`test/stash.test.js`), POC `poc/rc4-restorable-poc.mjs`.
- **Semantics:** park a large payload (tool result / fetched page / file dump) out of the context
  window keeping only the `id` handle; `get(id)` rehydrates verbatim, `forget(id)` evicts. A stash is
  **NOT memory** — never indexed, so recall can't surface it on any kind; **never auto-pruned** (survives
  the episode rolling-window prune, so restore always works); reachable only by exact id. First citizen
  of the "agent context" domain, separate from the searchable memory core (`mem`/`docs`).
- **LIBRARY API ONLY** — not yet wired to the CLI (`bin/litectx.js`) or MCP (`bin/litectx-mcp.js`).
  That's the obvious follow-up.
